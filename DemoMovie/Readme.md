# SDL_Demo_Phase2
------------------
* Phase1から機能を拡張
* 社外の情報（インターネット上のリアルタイム情報）をAndroidアプリで取得し、その後、車内の情報（車輌情報）に基づいてリアルタイム情報をSDL機器側に動的に内容を表示する方法を提示する 
  

## SDL_Demo_Phase2_1.mp4
------------------

| 時間 | 説明 |
| ---- | ------- |
|開始〜|infoボタン<br>接続したことのある車両情報を表示します|
|0:05〜 | RSSボタン<br>購読したいRSS情報を設定します(デモ用なので、1件のみ)|
|0:15〜 | 設定ボタン<br>Phase1で通知した項目の設定変更が行えます。<br>- 燃料残量を60%/40%/30%/25%で通知するように変更します<br>- タイヤ空気圧の通知、ヘッドライトの通知は変更しません|
|0:36〜|スマホをHU(ヘッドユニット)に接続します|
|0:42〜|HU上に表示されたアプリアイコンを選択します|
|0:47〜|タイヤ空気圧のデモ<br>※Phase1と同様の内容のため割愛|
|1:18〜|ヘッドライトのデモ<br>※Phase1と同様の内容のため割愛|
|1:40〜|燃料通知のデモ<br>設定画面で設定した値に燃料残量を指定すると、通知が行われます|
|2:27〜|HU(ヘッドユニット)との接続を終了します|
|2:30〜|再度infoボタンを選択して、接続した車両情報を確認します|
|2:41〜|再びスマホをHU(ヘッドユニット)に接続します|
|2:51〜|HU上に表示されたアプリアイコンを選択します|

## SDL_Demo_Phase2_2.mp4
------------------

| 時間 | 説明 |
| ---- | ------- |
|開始〜| RSSボタン<br>購読したいRSS情報が存在することを確認します|
|0:15〜| 車両停止状態にRSSの更新があれば通知を行います。<br>ここでの車両停止状態とは<br>  - 車速(Speed)が0であること<br>  - ブレーキペダルが踏まれていること<br>として定義しています<br><br>車両停止状態が一定時間続くと、RSSの更新情報を読み上げます<br>     ※デモ用に更新の有無に関わらず常にRSSの上位3件を通知するようにしています。|
